2. Architecture Violations (OOP & MVC)

ðŸ”´ Violation C: The "Inline Model" Anti-Pattern (Critical)

Location: src/controllers/CompetitionController.ts Defect: The file defines two Classes internally that are not exported or separated:

  1. class CompetitionRequestModel (Lines ~20-100)
  2. class RatingModel (Lines ~105-133)

Principle Violated: Separation of Concerns (SoC) and Single Responsibility Principle (SRP).

  • Controllers should only handle HTTP requests methods (get, post).
  • Models should define data access.
  • Embedding models inside controllers makes them:
    • Untestable: You cannot unit test the model without mocking the whole controller.
    • Unreusable: Other parts of the app cannot access RatingModel.
    • Unmaintainable: Bloats the controller file to 900+ lines.

Remediation:

  1. Create src/models/CompetitionRequest.ts.
  2. Create src/models/Rating.ts.
  3. Move the logic there.
  4. Import them in CompetitionController.

Location: src/controllers/CompetitionController.ts Defect: The controller contains heavy business logic for:

  • P2P Rating weighting calculations.
  • YouTube URL parsing.
  • Comment filtering logic. Principle Violated: MVC. Controllers should be "Thin". Business logic belongs in a Service Layer (e.g., CompetitionService). Remediation: Extract logic to src/services/CompetitionService.ts.
